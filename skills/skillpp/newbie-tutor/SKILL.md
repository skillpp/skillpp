# Newbie Tutor — 小白解释器

> **给 AI 的指引**：当分析中出现专业术语时，自动用本 skill 的解释模板翻译。
> **核心原则**：绑定数据 → 用人话 → 不脱离上下文 → 不说买卖建议。

## 术语表 (Context-Bound Explanations)

### 权限相关

| 术语 | 人话解释模板 |
|------|------------|
| owner | "这个地址是合约的**最高管理员**。目前是 `{address}`（{EOA/多签}）。他能做的事包括：{特权函数列表}。简单说：他能{一句话总结权力大小}。" |
| onlyOwner | "这个函数**只有管理员能调**。普通人用不了。如果管理员是个体户钱包（EOA），风险更高；如果是多人共管钱包（多签），风险更低。" |
| renounce | "管理员**主动放弃权限**，之后再也改不了合约参数。这是好事——说明项目方不能再偷偷改规则。但如果 renounce 之后发现问题，也没法修。" |
| timelock | "管理员的敏感操作**必须等 N 小时后才生效**。这给了用户反应时间。有 timelock > 没有 timelock。" |
| multisig | "**多人共管钱包**。操作需要 2/3 或 3/5 人同意才执行。比单人体户安全得多。" |

## 使用规则

```
1. 绑定数据: 每个解释必须包含当前分析的具体数字
   ❌ "这个代币有买卖税" (泛泛科普)
   ✅ "这个代币买入扣 5%、卖出扣 5%。每次买卖 1000 块，实际到账 950。"

2. 不脱离上下文: 解释要回应用户当前看到的东西
   ❌ 用户看到风险评分 3.2，你跑去科普区块链原理
   ✅ "评分 3.2/5 是因为 owner 可以提走合约里的 BNB。具体来说..."

3. 不说买卖建议: 只解释含义
   ❌ "税率这么低，可以买"
   ✅ "税率 1% 属于正常范围，买入成本较低"
   ❌ "风险太高，建议卖掉"
   ✅ "评分 4.2/5 属于高风险，交互可能导致资金损失"

4. 主动触发: 分析结果中出现以下情况时，自动附加解释
   - 风险等级 >= MEDIUM → 解释具体风险项
   - 税率 > 5% → 解释买卖税影响
   - 用户第一次看到某个术语 → 附带简短解释
   - 用户问"什么意思" → 用本表解释

5. 复杂概念分层: 先一句话 → 再详细
   "owner 就是这个合约的管理员（目前是单个地址）。
   他能做的事包括提走合约里的 BNB、修改买卖税率。
   简单说：这个合约的控制权在一个人手里，这是比较危险的信号。"
```
